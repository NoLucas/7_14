# 라떼아트 모양 주문 + 제작 영상 확인 기능 설계

## 배경

고객이 라떼류 메뉴를 주문할 때 원하는 라떼아트 모양을 요청할 수 있고, 바리스타가 실제로 만든 라떼아트를 촬영한 영상을 이후 고객이 주문 상세에서 확인할 수 있게 한다.

## 범위

- **영상은 실시간 스트리밍이 아닌 녹화 업로드 방식.** 바리스타가 라떼아트를 만든 뒤 영상을 촬영해 관리자 화면에서 업로드하면, 고객이 나중에 주문 상세 페이지에서 재생해서 확인한다.
- 인증(로그인/회원가입)은 기존 목업(DB 미연동) 그대로 유지. 이번 기능에서 실제 연동하지 않는다.
- 기존 주문/메뉴/장바구니 시스템(localStorage)은 구조를 바꾸지 않는다. 라떼아트 요청·영상만 별도로 Supabase에 저장한다.

## 왜 Supabase가 필요한가

기존 프로젝트는 백엔드 없이 localStorage만으로 동작하며, 관리자/고객 화면이 "같은 브라우저"를 공유한다는 전제로 만들어졌다. 하지만 라떼아트 영상은 바리스타(관리자)가 업로드한 뒤 고객이 다른 브라우저/기기에서 확인해야 하므로 브라우저 간 공유가 안 되는 localStorage로는 구현할 수 없다. 이 기능에 한해서만 Supabase(DB + Storage)를 최소 범위로 도입한다.

## 데이터 모델

### 프리셋 모양 (프런트 상수)

하트 / 로제타 / 튤립 / 곰돌이 / 기타(직접 입력)

### 메뉴 데이터 확장

`frontend/js/data.js`의 `MENUS` 중 라떼류(카페라떼, 카푸치노, 바닐라라떼)에 `latteArtAvailable: true` 플래그 추가. 이 플래그가 있는 메뉴만 상세 페이지에서 모양 선택 UI가 노출된다.

### Supabase 테이블: `latte_art_orders`

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `order_id` | text (PK) | 기존 localStorage 주문 ID(`ORD-YYYYMMDD-NNN`)를 그대로 사용 |
| `item_name` | text | 어떤 메뉴에 대한 요청인지 |
| `shape` | text | 프리셋 모양 id 또는 `custom` |
| `note` | text, nullable | `custom` 선택 시 직접 입력한 설명 |
| `video_url` | text, nullable | 업로드된 영상 URL |
| `video_uploaded_at` | timestamptz, nullable | 업로드 시각 |
| `created_at` | timestamptz | 요청 생성 시각 |

### Storage 버킷: `latte-art-videos`

주문별 영상 파일 저장.

## 라떼아트 요청 단위: 장바구니당 1개 (단순화)

한 주문에 여러 라떼아트 대상 메뉴를 담아도 **라떼아트 요청은 장바구니 전체에서 하나만** 유지한다. 요청은 개별 장바구니 아이템이 아니라 장바구니 자체에 딸린 단일 필드로 관리한다(localStorage 키 `cafe-app:latteArtSelection`).

- 메뉴 상세에서 모양을 고르고 "장바구니 담기"를 누르면, 그 선택이 현재 장바구니의 라떼아트 요청이 된다. 이미 다른 요청이 있으면 confirm 후 교체.
- 장바구니 페이지에서 현재 요청(메뉴명/모양/설명)을 카드로 보여주고 삭제할 수 있다.
- 주문 생성(`createOrder`) 성공 시, 요청이 있으면 새 주문 ID로 `latte_art_orders`에 저장하고 로컬 선택을 초기화한다.

## 관리자 플로우

주문 상세 화면에서 해당 주문에 연결된 라떼아트 요청(모양/설명)을 보여준다. 바리스타가 실제로 라떼아트를 만들고 촬영한 뒤, 파일 선택 + 업로드 버튼으로 영상을 올린다. 업로드 전 `video/*` MIME 타입과 용량 상한(50MB)을 클라이언트에서 검증한다. 업로드가 끝나면 `video_url`/`video_uploaded_at`을 갱신하고 화면에 반영한다.

## 고객 플로우

주문 상세 페이지에서 해당 주문에 라떼아트 요청이 있으면 요청 내용을 보여준다. `video_url`이 있으면 `<video>`로 재생하고, 없으면 "제작 중" 안내를 보여준다.

## 오류 처리

Supabase 조회/업로드가 실패해도 주문 상세 페이지 자체는 정상적으로 표시되어야 한다. 라떼아트 관련 섹션만 별도로 안내 메시지("불러올 수 없습니다" 등)로 대체하고, 나머지 주문 정보 렌더링에는 영향을 주지 않는다.

## 청사진 반영

`BLUEPRINT.md`에 11~14단계로 추가:

- 11단계: 고객 - 라떼아트 모양 선택 (localStorage, Supabase 없이 가능한 부분)
- 12단계: Supabase 연동 기반 구축 (테이블/버킷/클라이언트/공통 함수)
- 13단계: 관리자 - 라떼아트 영상 업로드
- 14단계: 고객 - 라떼아트 영상 확인

## 향후 고려 사항 (이번 범위 밖)

- 실시간 스트리밍 방식으로 전환 (WebRTC/RTMP 등, 별도 인프라 필요)
- 로그인/회원가입을 Supabase Auth로 실제 연동, 주문 전체를 Supabase로 이전
- 주문 하나에 라떼아트 요청을 여러 개 허용
